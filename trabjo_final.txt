Si el usuario no es root deberá mostrar las salidas de a, b y c
Caso contrario deberá ejecutar todos los chequeos. 
Uso de opciones --help 
		-d disk 
		-t time 
		-l logfile 


Encabezado: 
####################################
# hostname 			   #
# estampilla de tiempo 		   #
# archivo donde se reporta el log  #
####################################
	

Resúmen de ejecución:
########################################
RESUMEN
########################################
	Total de chequeos realizados
	Total de exito. 
	Total de errores encontrados.
	Total de advertencias. 

Formato de salida de exito, error y advertencia por chequeo. 
*** EXITO:       mensaje
*** ERROR:       mensaje
*** ADVERTENCIA: mensaje



Verificaciones:
---------------
Verificación del servicio de ntp:
	Si esta no esta ejecutandose entonces reportar como advertencia
	ADVERTENCIAS. 
	Si esta ejecutando y sincronizando entonces  EXITO, SINO ERROR. 

Verificación de sistemas montados:
Verificar si todos los sistmas de archivos en /etc/fstab se encuentran montados. 
	Si todos estan montados, entonces no se muestra salida.  
	Si alguno no se encuentra montado entonces ERROR.
	

Memoria :
 - Verificar cuanta memoria RAM tiene el sistema, cuanta en uso y cuanta libre.
 - Mostrar los 5 procesos que mas memoria RAM utilizan.

CPU :
 - Verificar el uso de CPU. Cuanto se está utilizando para procesos, cuanto
   para el sistema, cuanta espera hay de E/S, y cuanto % de CPU hay disponible.
 - Mostrar los 5 procesos que mas CPU están utilizando.

RED :
 - Mostrar la configuracion de la placa de red principal. Velocidad (100Mbit/s,    1Gbit/s, Full duplex, half duplex).
 - Verificar el uso del ancho de banda de la red. Cuánto está en uso y cuanto
   libre.
 - Mostrar los 5 procesos que más ancho de banda de la red están utilizando.

Disco :
  - Mostrar la configuracion del disco principal del sistema y del sistema
    de archivo raiz.
  - Mostrar los 5 procesos que mas E/S están realizando a disco.
